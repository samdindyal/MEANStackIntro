Prepare Web App for Heroku

Before we add our web app to Heroku, we need to add something called a Procfile. The Procfile tells Heroku which source file it needs to run to launch your app.

Fire up your text editor and enter the following and save the file as Procfile in your project's root directory: 

  web: node server.js

Setting up Heroku Toolbelt

For the sake of simplicity, please stay in the root directory of your project. Now that you've got your web app, we can deploy it to Heroku. The first thing we need is the Heroku Toolbelt.

Now that you've got Heroku Toolbelt installed, you'll need a Heroku account. You can sign up below:

https://signup.heroku.com

Once Heroku Toolbelt has been installed, from the root directory of your project, run the following: 

  $ heroku login
You will then be prompted to log into your Heroku account. Enter your credentials to continue.

Creating Your Heroku Web App

Before turning our web app into a Heroku app, we need to initialize our project folder as a git repository. This can be done with the command below: 

  $ git init

Once our git repository has been successfully initialized, use the command below to create a new Heroku web app: 

  $ heroku create <appname>
Where you replace appname with the name of your app. The appname value will be used as a part of your web app's url. Your web app's url will be as follows:

http://<appname>.herokuapp.com

Deploying to Heroku

Now that we're all set up, we can add our web app to Heroku. Please execute the following, line by line: 

  $ git add --all
  $ git commit -m "First commit"
  $ git push heroku master

View Your App Live!

Once you have deployed your site to Heroku, you can view it by manually typing in the URL into your browser or, from your project's root directory, you can run the following: 
  $ heroku open
