In our experience writing this web app, we have noted a few things.

The Learning Curve

Because the MEAN Stack is predominantly JavaScript, with a little HTML and CSS here and there, the learning curve was not nearly as steep because there was only one language used for the server and client sides. Along with that, JavaScript is a widely supporting language with a huge community making it easy to find support online and, in our experience, JavaScript is a relatively easy language to learn.

Callbacks

Unlike a lot of event driven programming, node.js, which handles running the web server for a MEAN Stack web app, runs on callbacks. With callbacks, a functionA() is passed to functionB(). functionA() will call functionB() after it has finished executing. This is different to AngularJS which uses promises instead, where functions are chained onto another. Though some programmers prefer to use promises, call backs are substantially more efficient. While writing this app, we found it somewhat strange switching between callbacks on the server side with node.js and promises on the client side, AngularJS.

Heroku and Ports

Initially when we deployed our web app to Heroku, it kept crashing out. We were stumped as to why our app wouldn't run since it did perfectly locally. The problem we had was a confliction between Heroku and the static port we defined in our code for our app. When Heroku runs an app, it chooses a port and runs it there. The port we specified was used, but Heroku didn't know that and it waited until there was a timeout. We fixed this by using process.env.PORT || 3000 as our code variable instead. process.env.PORT is a Heroku environmental variable container the port of which Heroku is using to run an app. The || operator guarantees that a port is defined by providing a backup should the environmental variable be unaccessible, 3000. The port variable is then passed into the .listen() method in express.js to start the web server. Our web app started working immediately after we implemented this change.

Conclusion

In conclusion, we believe that the MEAN Stack is great for building dynamic apps with scalability. It eliminates the need for another server side language and it has an enormous community and libraries which are still quickly growing.
